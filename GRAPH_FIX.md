# Computation Graph Fix

## Problem

The demo was manually constructing the computation graph using regex-based code parsing (`compile_parse.py`), when skrub provides an authoritative native graph via `DataOp.skb.draw_graph()`.

## What Was Changed

### Backend (Already Working)
- `skrub_graph_runner.py` already calls `DataOp.skb.draw_graph()` during execution
- `execute_stream.py` already emits `skrub_graph` SSE event with the SVG
- No backend changes needed

### Frontend Changes

1. **CodeGenDemo.tsx**
   - Changed from `skrubGraphSvg={null}` to `skrubGraphSvg={skrubGraphSvg}`
   - Graph now displays skrub's native graph after execution
   - Static preview shown before Run, replaced with skrub graph after

2. **GraphPanel.tsx**
   - Updated header to show "(skrub native)" when displaying skrub's graph
   - Changed description text:
     - Before Run: "Static preview · Run to see full skrub graph with all operations"
     - After Run: "DataOp graph from execution · Shows all operations and data flow"
   - Added `skrub-graph-container` CSS class

3. **index.css**
   - Added CSS styling for skrub's SVG to match light theme
   - Styled text, nodes, edges, and arrows with zinc-900 colors
   - Made nodes clickable with hover effects

4. **Tests**
   - Updated GraphPanel test to match new UI text
   - Added test case for skrub native graph header text

### Documentation

- Updated `compile_parse.py` docstring to clarify it provides "static preview" only
- Emphasized that skrub's native graph is authoritative after execution

## Benefits

1. **Authoritative Graph**: Skrub's graph shows the true DataOp computation graph
2. **Complete**: Shows ALL operations (Var, SubsamplePreviews, GetItem, semantic operators, etc.)
3. **Accurate Data Flow**: True dependencies, not document-order approximation
4. **No Maintenance**: No need to keep regex patterns in sync with skrub/sempipes APIs
5. **Rich Metadata**: Nodes include tooltips with source file/line locations

## How It Works

### Before Execution (Static Preview)
- Parse code with regex to show approximate graph
- User sees rough idea of pipeline structure
- Header shows: "Computation graph"

### After Execution (Skrub Native)
- Pipeline runs in subprocess via `skrub_graph_runner.py`
- Skrub's `DataOp.skb.draw_graph()` generates authoritative SVG
- SVG streamed to frontend via SSE `skrub_graph` event
- Graph replaced with skrub's native visualization
- Header shows: "Computation graph (skrub native)"

## Skrub Graph Features

Skrub's graph (generated by graphviz) includes:
- **All DataOp operations**: Var, SubsamplePreviews, GetItem, semantic operators
- **True data flow**: Edges from producer to consumer operations
- **Colored nodes**: X nodes (blue), y nodes (colored), regular operations
- **Source locations**: Tooltips show file and line number where each operation was defined
- **Nested structure**: Double borders for Var nodes
